# TodoList 项目需求分析文档

**日期**: 2026-04-15
**项目名称**: TodoList
**项目类型**: 全栈 Web 应用（前后端分离）

---

## 一、项目概述

TodoList 是一个任务管理应用，支持用户注册登录、管理自己的任务清单，提供任务分类、标签、优先级、截止日期等高级功能。

---

## 二、功能需求

### 2.1 用户认证

| 功能       | 描述                          |
| ---------- | ----------------------------- |
| 注册       | 用户名、邮箱、密码注册        |
| 登录       | 邮箱+密码登录，返回 JWT Token |
| Token 刷新 | Access Token 过期后刷新       |
| 登出       | 客户端清除 Token              |

### 2.2 分类管理（层级目录）

| 功能     | 描述                                      |
| -------- | ----------------------------------------- |
| 创建分类 | 支持指定父分类（顶级分类 parent_id=null） |
| 查看分类 | 获取当前用户的全部分类树                  |
| 编辑分类 | 修改分类名称                              |
| 删除分类 | 删除时处理子分类（级联或提升）            |

### 2.3 标签管理

| 功能     | 描述                   |
| -------- | ---------------------- |
| 创建标签 | 输入标签名称           |
| 查看标签 | 获取当前用户的所有标签 |
| 编辑标签 | 修改标签名称           |
| 删除标签 | 解除标签与任务的关联   |

### 2.4 任务管理

| 功能     | 描述                                         |
| -------- | -------------------------------------------- |
| 创建任务 | 标题、描述、所属分类、优先级、截止日期、标签 |
| 查看任务 | 按分类/标签/状态筛选，支持分页               |
| 编辑任务 | 修改任意字段                                 |
| 标记完成 | 切换任务完成状态                             |
| 删除任务 | 软删除或物理删除                             |

### 2.5 任务字段

| 字段         | 类型         | 说明                 |
| ------------ | ------------ | -------------------- |
| id           | INT          | 主键                 |
| title        | VARCHAR(255) | 任务标题             |
| description  | TEXT         | 任务描述（可选）     |
| category_id  | INT          | 所属分类（外键）     |
| priority     | ENUM         | high / medium / low  |
| due_date     | DATETIME     | 截止日期（可选）     |
| is_completed | BOOLEAN      | 是否完成，默认 false |
| created_at   | DATETIME     | 创建时间             |
| updated_at   | DATETIME     | 更新时间             |
